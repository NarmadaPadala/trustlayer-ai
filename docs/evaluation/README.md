# TrustLayer AI Evaluation Package

This folder contains the production-style evaluation evidence for TrustLayer AI, a multi-agent Python code review system.

The evaluation package is organized around the AI engineering loop:

1. Define measurable outcomes.
2. Build a golden dataset.
3. Instrument the agent workflow.
4. Run a baseline evaluation.
5. Analyze failures.
6. Improve the system.
7. Re-run the same benchmark.
8. Package results for portfolio review and interviews.

## Result Summary

| Run | Pass | Conditional | Fail | Critical Recall | Routing Accuracy | Human Review |
|---|---:|---:|---:|---:|---:|---:|
| Baseline | 26 | 7 | 7 | 57.1% | 85.0% | 57.1% |
| Improved | 33 | 7 | 0 | 100% | 100% | 100% |

The main baseline failure mode was missed critical security findings. The improvement focused on Security Agent heuristics, prompt specificity, structured output expectations, and regression tests for known failure classes.
